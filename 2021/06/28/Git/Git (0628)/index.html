認識Git

  • subversion(SVN) ：集中伺服器 （中央統一的概念，但壞處是沒有網路時就無法使用）
  • stackoverflow
  • 什麼事git: 分散式版本控制系統，是為了更好管理Linux內核而開發的。
  • Git優點是：可以了解檔案內容的變化過程/掌握誰寫的內容/支援遠端及本地（不需網路也可以使用）操作/共享
  • 誰發明Git: Linus （其實只是內容檔案/////）
  • git 和 githut 的不同
  • 不要害怕指，仔細閱讀錯誤

Git

  • 先設定資料 ： git config –list (無跑資料就是無設定，有的話會有訊息)
  • 假裝我會，直到真的會為止

Git快捷鍵 （專業工程師必備）

  • 鍵盤“Ｑ”是結束
  • git config –list | grep (指定的搜尋名稱，就會把所有指定名稱紀錄列出來)
  • ls ,ls -a
  • 清畫面 ＝ fclear ＝ control+L
  • 刪除前單字 ＝ control+W
  • 刪除整行 ＝ control+Ｕ
  • 搜尋或歷史紀錄 ＝ control+Ｒ

操作Git

新建立

  1. cd /tmp
  2. mkdir（建立） (新資料夾名稱)
  3. git init( 授權給GIT控制)
  4. git status

創造＋存擋

  1. touch index.html
  2. git add index.html （檔案存檔）
  3. git status
  4. git commit -m “建立資料(打紀錄內容)”
    （ps:好的工程師對於commit的精細度是很重要的。也就是在哪建立commit點。）

ps: 簡單理解VIM

檢視紀錄

  1. git log / (建議可使用圖像軟體看)
    (與上面意義相同，但可以更看細節的代號)git log –oneline

把檔案刪除/救回來
git rm *html (全刪除 git add –all)
git rm -r config
git restore
(git mv直接刪除 -有風險)

這檔案誰寫的

git blame index.html(要查的資料)
檢視特定檔案
git log -p hello.html(檔案名稱)

checkout 指令：轉換

如果從他人的git內容下載（例：老師給的git-example包)
到自己的電腦後要再次進行作業，
1.需要再次實施 git init對吧？

A:先判斷載入的檔案是否已有分支。（通常有綠色分之的都代表已有了）

2.皆上述；是因為是由他人不同電腦的git載入，所以我們要再次執行git init拿到控制權，這樣的邏輯意思對嗎
A:邏輯方向對。但如下老師說的，『建議不要在一個專案內放另一個專案的版控，』

Q2續上；

老師說 『建議不要在一個專案內放另一個專案的版控，』
