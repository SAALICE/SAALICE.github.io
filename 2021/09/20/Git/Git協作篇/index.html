Git 協作篇

發表於2021-09-20|更新於2021-10-03
|閱讀量:

git協作

協作篇：

0.先確定是否有clone過專案下來。（製作時都開新分支製作）

＝＝＝＝＝＝＝＝＝＝＝

假如現在，我站在 『 本機 』 feat/oauth 分支身上：

==================
1.拉新分支

git fetch origin develop
(確認本機無此 dev 分支才需要使用 fetch)

2.切換到主分支

git checkout develop
（切記要用分支 checkout分支，若目前於“點”上，請先切換到新分支。）

3.更新分支

git pull origin develop
（如果是在原先 feat/oauth 分支上直接進行 git pull origin develop， 會造成你把 develop 直接接到 Ａ分支上。 變成錯誤rebase 的效果在Ａ 分之身上。）

4.切回操作分支

git checkout feat/oauth
(回到你要的分支身上)

5.進行 rebase

git rebase develope (會顯示，已是最新的)
(這動作目的是，為了再次確認我要的這分支是否真的已更新到最新的上了。)

＝＝＝＝＝＝＝＝＝＝＝
✏️發生衝突 ->存檔提交
(可能會有衝突，在本機解除，才利於遠端上運作。)
衝突完成後，執行：

隨時要記得使用 git status 進行提交的確認 。

git status
git add . /
git commit -m “”
git rebase –continue

＝＝＝＝＝＝＝＝＝＝＝

將完成的分支先推上 develop！！！

<超級重要！了解什麼是 rebase 後產生的內容與方法使用。>
6.更新（推上）遠端分支: X

git push origin feat/oauth
這裡推上 develop ，是 “ 失敗“ 的！！！
原因是當你進行 rebase 後，本機的 feat/oauth 已經和當初的 feat/oauth 所在的點不同了。因此它提醒你目前是落後的，且會是 feat/oauth -> feat/oauth (non-fast-forward)

7.更新（推上）遠端分支

git push origin feat/oauth -f
(當你非常肯定、確定於此分支是最新且是你要的，才可以使用 -f 進行強推)



其他必要指令相關

git remote -v: 查看遠端連結（URL）
git remote add((origin)(git@~.git))
git remote remote (origin): 移除遠端連結
git rebase –quit 強制rebase離開(壞掉)
