1.CSRF的驗證手序：

請記住，config.omniauth將 omniauth 提供程序中間件添加到您的應用程序。這意味著你應該不會再次添加此供應商的中間件config/initializers/omniauth.rb因為他們會互相衝突，並導致始終未能驗證。

2.安裝套件

3.至 routes 新增方法

config/routes.rb

4.手動新增一個 callbacks_controller

app/controllers/users/omniauth_callbacks_controller.rb

5.user.rb中新增omniauthable 模組

需要有 provider 提供者 / uid 用戶介面 此兩欄位。
(這兩個欄位是用來記錄GitHub的資訊!)

6.開啟 devise 三方的設定
app/config/initializers/devise.rb

7.開啟 rails s ：出現 github 連結按鈕！

8.至 gitgub 註冊三方後會得到 Client_ID 與 Client_Secret

一般來說，流程到此結束即可在 {http://localhost:3000/} 環境下使用了！

但注意！如果會公開上傳至公開場域！
==務必將鑰匙上鎖 —>建立環境變數。==

推薦 “ rails 內建的 credentials.yml.enc”
或 “ figaro 套件 “ 來存放鑰匙。
並且成為 ==.gitignore== 的（上傳）忽略清單。

可參考此篇

credentials 解密參考網址

PS:
Devise 中只有三個 action 允許設定的欄位通過傳到 model，
分別是：

sign_in ( Devise::SessionsController#create )
只允許 authentication keys 通過( ex: email、account)

sign_up ( Devise::RegistrationsController#create )
只允許 authentication keys、password、password_confirmation

account_update
(Devise::RegistrationsController#update )
只允許 authentication keys、password、password_confirmation、 current_password
