**當我們一起出發，”this”你是誰？ **

記住這張圖！ 等等你一定用得上的！

很多人説 this 就像是function中的黑魔法之一，
難搞又麻煩，還喜歡變來變去，看似親民、高級卻易使人迷惑。
但總在關鍵時候讓人捏把冷汗。 要想駕馭它就必須先擊破它！
這裡;我想表達的是 用老招：

『反過來說。』

如果你能先掌握this的改變條件，
聰明的你就一定會有很多方法能破解和提升的！
今天會從多個例子與五倍課堂上高見龍老師教授的秘技限制裡一起去分析。

其實this有兩種：

第一種偏向使用new方法產生的this物件導向。（本文站不討論）
第二種透過function中的this物件導向。（今日討論方向）

那麼回到主題開始;
this在function中是什麼呢？
就像我問你：『你是誰』
this 就如同這個你一樣，它是一個存在的，值。
這值由被呼叫的函式來決定。它不能在執行期間被指派，每次函式呼叫調用的值也可能不同。（出自MDN定義）

針對上述的第二種解釋，可以想像成
在瀏覽器底下 ： window
在node.js 底下 ： global
但這只是同個角色在不同背景下的不同稱呼，它們指向的this都是同一個人。

記住; this是取決於誰呼叫他，而非誰在哪、呼叫他。

但當this出現在下列條件時，this的值就會被改變。
這就是： 反過來說 的條件 （by高見龍老師教學提供的秘技～真的是相當好記！又具可觀性！推～）
