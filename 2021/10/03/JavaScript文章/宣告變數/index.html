用JavaScript宣告變數: Var / let / const

在每個語言中都有自己宣告語言的方式，
當然JS也不例外;

在其他文章裡，你可以了解關於ES6中有新增關於強化let 與 const的使用，來替代var在開發使用上容易造成全域變數污染整個環境的影響。
雖然多數企業已開始使用前兩者居多，但你不能不知道的三種宣告差別，就讓我們用以下例子來簡單說明吧！
(註：若想延伸了解關於ES6說明，請點選我)

威～～差點離題了！！（拉繩！！）!
今天我們專注在宣告變數上，
以宣告變數來說，
我們可以比喻成用『嚴謹強度』來作為三者的區分，但分別有各自的定義及作用域。

（作用域：最寬鬆）
var:宣告可適在全域內進行任何的變數宣告，或是可用在 function 而不管該區塊範圍。

（作用域：有條件使用）
let:宣告只能在目前區域、階段或表達式中作用的變數。
✏️可視範圍可以是全域內，或是在宣告的區域內出現。
✏️接上; let 只能活在{ }宣告的區域內使用。離開便失效。

（作用域：全在控制中，且不能重複）
const: 所宣告的變數，具有區塊可視範圍。常數不能重複指定值，也不能重複宣告。
✏️可視範圍可以是全域內，或是在宣告的區域內出現。
✏️作為一般全域常數 => 不能重複定義或命名
✏️作為區域中物件內容的參數 => 可以變更（但通常請避免！）

但是;當三個都在同個環境裡出現時，作用域效能越大的（越嚴謹的）越優先，也就是const > let > var 。

（如欲更深入了解各自定義解說，可參考MDN分別出處 ）
