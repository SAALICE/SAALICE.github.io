用想像例來說明JavaScript物件導向

你記得哆拉a夢都搭乘時光機 ..咻咻得去哪嗎～
它可以從某個時間點往前或往後追到另個時間點。
你可以想像成眼前你所看到的每個實物個體都是由過去某樣東西從原型模具而組成你眼前的模樣。

再想像下例這張圖內的機器如果實際在眼前的模樣，從過去跟現在會是怎麼產生。

將這畫面再放到主題後，
想像下列這句;
『JavaScript中物件導向就像從一個（類別）物件演化成一個程式設計。』
有辦法想像出這個畫面嗎～
沒關係，我們先來一步步的解釋這句話！
首先;角色介紹：

✏️（類別） => 最原型的模型樣式物件 （像健身房裡常說的吃原型食物的”原型”同義），所有物件都從這開始分類建造出來。

✏️ 物件 => 由（類別）繼承、分類原有的樣子及功能，再使用建構子 (Constructor) 函式建立成物件實體，將這物件實體化。
—（簡單說就是從類別模板再實體化出一個新的物件。）

✏️ 特殊物件 => 原理同上，只是加入別於類別模板的個人特色或新功能後，一樣透過建構子建立，並實體化成新物件。

✏️ 程式設計 => 物件透過一種方法，稱之為建構子函式（constructor function）的特殊函式，定義物件與功能。並透過集合核心稱為原型鏈（Prototype chain）練成一種鏈結。

((👩🏻‍🦰 若你想深入瞭解最原型的模型樣式物件組成，或是更深層物件的過去/現在 如何鏈結組成：__proto__與protoye之間關聯） 可點我 。 出處MDN詳細介紹。

好的；
接下來，你一定開始想像以上角色是如何透過函數串在一起的？
我們就再往下一點，

『一般來說所謂物件（類別物件/特殊物件）都是透過原型來做新增模板的實體化。而這些物件都具有：屬性 ＋ 行為。物件產生後透過建構函示來串成系列程式設計，而這方法我們都會通常會使用 obj（）或是 new 這兩種方法來做新增複數。

以下我們就透過範例的過程來介紹吧！

首先;這是範例中共用的模板（已經實體化後的物件）及現狀：
『工廠裡的馬達是由零件們串接而組成。』

現在要讓工廠裡的馬達以原型複製並加入一種新工具製造。
兩種方法：

第一種：
Object.create() 指定其原型物件與屬性，創建一個新物件。

這裡需要細細品嚐的是最後兩行。從這裡的答案推回去，你會發現：
1.當執行第18行時，會先印出factory裡有的內容。
2.接下來執行第19行時，會印出新產出machineTools的內容。
3.由此二你可得知，machineTools是從模板factory透過原形鏈經由Object.create後出來的，但不具有原來machine功能的。可在之後你卻可以任意直接呼叫machineTools（），拿到裡面的新工具。而不影響原來的模板。

第二種 ：
new : 創建一个用户定義的對象類型的實例或具有構建函數的內置對象的實例。

這裡複製原型後新增新工具的方法比較簡單。
